# 链上交易模拟平台 PRD / PH / TodoList

## 1. 项目背景

本项目目标不是直接复刻 Hyperliquid，而是构建一个**以策略模拟、自动交易研发、用户训练为核心**的高拟真交易平台基础设施。

平台重点包括：

- 不只依赖 K 线，而是记录更接近实盘的执行参数
- 支持策略在线模拟与横向比较
- 支持未来接入自动交易系统
- 支持未来用户快速理解交易逻辑、保证金逻辑、爆仓逻辑
- 在架构上预留区块链结算/存证/账户映射能力

---

## 2. 产品目标

### 2.1 核心目标
1. 构建一个高拟真交易模拟内核
2. 支持订单、仓位、保证金、强平、回放
3. 在 PH1 做出一个可用的交易模拟内核 + 基础 Web 原型
4. 支持记录非 K 线参数，用于未来优化自动交易
5. 为未来链上化、策略竞赛、AI 分析打基础

### 2.2 非目标
第一版不追求：
- 真正的去中心化撮合
- 高频链上 orderbook
- 多链兼容
- 完整合规体系
- 高并发海量用户承载
- 完整生产级安全审计

---

## 3. 用户类型

### 3.1 平台研发者
- 研究交易平台底层逻辑
- 研究模拟与实盘差异
- 为未来自动交易系统准备基础设施

### 3.2 策略研究者
- 在线运行策略
- 比较不同策略表现
- 查看成交、滑点、爆仓、资金占用等参数

### 3.3 新用户 / 学习用户
- 通过模拟盘快速了解交易所规则
- 理解保证金、杠杆、爆仓、手续费

---

## 4. 产品定位

本平台定位为：

**链上可扩展的高拟真交易模拟平台**

它的本质不是普通回测工具，也不是传统交易所，而是：

- 交易模拟内核
- 自动交易实验平台
- 策略比较平台
- 用户训练平台

---

## 5. 核心能力

### 5.1 行情层
- 接入真实行情
- 支持 tick / bid / ask / spread
- 支持 K 线生成
- 支持历史回放行情

### 5.2 订单层
- 市价单
- 限价单
- 撤单
- 订单生命周期管理
- 部分成交
- 拒单原因记录

### 5.3 成交层
- 基于价格和规则模拟成交
- 模拟滑点
- 模拟延迟
- 模拟 spread 扩张

### 5.4 仓位层
- 开仓 / 加仓 / 减仓 / 平仓
- 平均持仓价计算
- 已实现盈亏
- 未实现盈亏

### 5.5 保证金与风险层
- 初始保证金
- 维持保证金
- 杠杆倍数
- 风险率
- 强平价计算
- 强平执行

### 5.6 账户层
- 模拟余额
- 可用余额
- 已占用保证金
- 资金流水
- 手续费记录

### 5.7 策略层
- 多策略并跑
- 独立资金池模式
- 共用资金池模式
- 策略标签与版本管理

### 5.8 回放与复盘层
- 回放每一笔订单
- 回放成交与仓位变化
- 回看爆仓原因
- 输出策略表现报告

### 5.9 AI 分析层
- 自动总结策略优缺点
- 自动解释交易行为
- 自动归纳策略适用市场环境
- 自动识别模拟参数缺失

### 5.10 链上扩展层
- 钱包登录预留
- 账户映射预留
- 关键快照上链预留
- 比赛结果存证预留

- AI 分析能力
- 链上登录与链上存证实现

## 8. 成功指标

### 8.1 产品层
- 能完整模拟一笔订单从下单到平仓
- 能正确计算保证金与强平
- 能完成单账户单品种模拟
- 能输出基础复盘结果
- 能通过 Web 页面完成操作和观察

### 8.2 技术层
- 订单状态无明显错乱
- 仓位与账户状态一致
- 回放结果可复现
- 核心逻辑具备测试覆盖

### 8.3 用户层
- 用户能快速理解交易规则
- 用户能看懂为何成交 / 为何爆仓
- 用户能比较多个策略

# 9. PH 阶段规划

## PH1：交易模拟内核 + 基础 Web 原型
目标：先做出一个可用的交易模拟内核和可操作的基础 Web 原型，把交易世界的核心状态机做稳

范围：
- 行情输入
- 订单状态机
- 成交逻辑
- 仓位计算
- 保证金计算
- 强平逻辑
- 资金流水
- 事件持久化
- Replay 能力
- API / WebSocket 基础验证
- 基础 Web 原型

交付物：
- 可运行 Trading Core
- 单元测试
- 核心事件流文档
- 可操作的 Web 原型
- `docker-compose` 本地开发环境

---

## PH2：策略实验平台
目标：在 PH1 稳定后，支持多策略在线模拟与比较

范围：
- 多策略并行运行
- 独立资金池 / 共用资金池
- 策略版本管理
- 策略结果比较
- 基础排行榜

交付物：
- 策略实验面板
- 策略对比报告

---

## PH3：AI 分析增强
目标：在结构化数据稳定后，增加 AI 总结和解释能力

范围：
- AI 自动总结策略优劣
- 自动解释交易行为
- 自动解释爆仓原因
- 自动识别缺失参数

交付物：
- AI 分析面板
- Prompt 与评估基线

---

## PH4：链上扩展与存证
目标：引入链上能力，但不强制全链上

范围：
- 钱包登录
- 账户映射
- 关键快照上链
- 比赛结果 / 回测结果存证
- 风险参数版本上链预留

交付物：
- 链上扩展模块
- 合约草案
- 链下/链上边界文档

---

## PH5：平台强化与用户训练
目标：面向更多真实用户和训练场景

范围：
- 用户教学引导
- AI 交易解释助手
- 策略比赛
- 复盘系统增强
- 更高拟真的执行环境参数

交付物：
- 用户训练模式
- AI 教练模式
- 更完整的策略竞赛雏形

# 11. 技术建议（首版）

## 推荐技术方向
- 整体语言：TypeScript
- 本地编排：docker-compose
- 后端：Node.js + Fastify
- 前端：Next.js
- 数据库：PostgreSQL
- 实时推送：WebSocket
- ORM：Prisma
- 测试：Vitest
- AI：OpenAI API（非 PH1 必需）
- 链上扩展：先保留接口，不在第一版深做

# 12. 第一版上线定义

满足以下条件即可视为第一版上线：

1. 可接入真实行情
2. 可完成模拟下单和撤单
3. 可维护仓位和账户权益
4. 可计算保证金和强平
5. 可输出基础回放结果
6. 可通过 Web 页面操作
7. 可通过 WebSocket 推送关键状态更新
8. 核心逻辑具备基础测试
9. 可通过 `docker-compose` 在本地运行
